## ğŸ¯ Design Principles

### 1. Branch Context First
- Current branch always visible in header
- All data filtered by selected branch
- No cross-branch data leakage

### 2. Clear Visual Hierarchy
- Important actions in primary blue
- Warnings in yellow/orange
- Errors in red
- Success in green

### 3. Progressive Disclosure
- Basic info shown first
- Details revealed on interaction
- Admin features hidden from non-admins

### 4. Immediate Feedback
- Loading states for all async operations
- Success/error toasts
- Optimistic UI updates

## ğŸ¯ Key UX Decisions

### 1. Why Tabs Instead of Separate Pages?
- **Faster navigation** - No page reload
- **Context preservation** - Branch selection persists
- **Better overview** - See all features at once

### 2. Why Inline Forms?
- **Reduced clicks** - No modal dialogs
- **Better context** - See list while editing
- **Easier cancellation** - Just close form

### 3. Why Status Badges?
- **Quick scanning** - Color + icon + text
- **Universal understanding** - Visual language
- **Accessibility** - Not color-only

### 4. Why Confirmation Dialogs?
- **Prevent mistakes** - Critical actions confirmed
- **User confidence** - Know what will happen
- **Undo prevention** - Some actions irreversible
